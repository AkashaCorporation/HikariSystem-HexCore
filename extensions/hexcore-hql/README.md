# HexCore HQL (Helix Query Language)

**Semantic Pattern Matching Engine for HikariSystem HexCore**

HQL (Helix Query Language) is the behavioral analysis layer for the HexCore IDE. Instead of relying on fragile, byte-level pattern matching (like YARA or FLIRT) that breaks with minor compiler changes or basic obfuscation, HQL queries the **Abstract Syntax Tree (AST)** generated by the Helix MLIR decompiler. 

Bytes change. Opcodes change. Registers change. **Math and logic do not.**

## Philosophy: Intent Over Syntax

Traditional reverse engineering tools scan for sequences of bytes (e.g., `55 8B EC...`). HQL scans for semantic structures. By operating on Helix's heavily optimized C-AST (post-Dead Code Elimination and Copy Propagation), HQL can identify complex algorithms, crypto routines, and obfuscation VMs regardless of the underlying architecture or compiler optimization level.

## Architecture

1. **Helix Flatbuffer Export:** The `hexcore-helix` engine compiles LLVM IR into a clean C-AST and serializes it via Flatbuffers (`helix_ast.fbs`).
2. **HQL Matcher:** This TypeScript engine hydrates the AST in memory and evaluates HQL JSON-based signatures against the tree nodes.
3. **Pipeline Integration:** Matches trigger `onResult` actions in the HexCore automation pipeline (e.g., auto-initiating Unicorn emulation when a specific decryption loop is detected).

## Example Signature (AST-Sig)

Detecting a custom CRC32/XOR algorithm inside an obfuscated loop:

```json
{
  "target": "function",
  "name": "Suspicious_XOR_Loop",
  "match": {
    "type": "CForStmt",
    "body": {
      "contains": "CBinaryExpr",
      "operator": "XOR",
      "operands": [
        { "type": "CVarRefExpr" },
        { "type": "CIntLitExpr", "value": "0xEDB88320" } 
      ]
    }
  },
  "confidenceBonus": 40
}
```

## Quick Start

This engine is built with strict TypeScript for high-performance V8 execution. Zero external heavy dependencies.

### Installation

```bash
npm install
```

### Build

Compile the engine into the `dist/` folder:

```bash
npm run build
```

### Running the Smoke Test

Validate the Semantic Matching logic against a mocked C-AST (XOR loop detection):

```bash
npm run test
```
